# bachelor thesis docs

dedicated thesis-planning workspace for `zalem`.

this folder keeps the thesis-specific material separate from the product/system plans in the root `docs/` folder.

---

## what is here

- `supervisor-package.md` — the best single file to show a supervisor: final title, abstract, research questions, hypotheses, methodology, and one-page proposal
- `current-direction.md` — the recommended thesis framing for `zalem`
- `evaluation-plan.md` — concrete evaluation design: baseline vs AI conditions, review-summary/comparison metrics, and sample questionnaire items
- `thesis-outline.md` — chapter-by-chapter thesis structure
- `titles-and-abstracts.md` — final title candidates and abstract drafts
- `thesis-mvp.md` — thesis-safe MVP scope and execution plan
- `other-thesis-ideas-2026.md` — 10 unrelated CS thesis ideas for 2026 with practical LLM integration

---

## how to use this folder

if you continue with `zalem`, read in this order:

1. `supervisor-package.md`
2. `current-direction.md`
3. `evaluation-plan.md`
4. `thesis-mvp.md`
5. `thesis-outline.md`

7. `other-thesis-ideas-2026.md`

## recommendation

for now, the best direction is:

> **build `zalem` as a trustworthy, reactive-first AI shopping advisor and evaluate it as a human-centered decision-support system, not just as an app**

that gives you:

- a real product artifact
- a modern LLM angle
- strong trust-building features through review summarization and AI-assisted comparison
- a believable CS thesis argument
- a strong demo
- room for evaluation, which is what makes it academically serious
